## 1. 背景与问题定义

### 1.1 背景

任务管理工具已经是一个成熟赛道，市场上有 Todoist、Things 3、Notion、TickTick 等大量成熟产品。但 AI 大语言模型的普及带来了一个新的机会：**用自然语言驱动任务管理**，彻底消除录入摩擦。

现有工具的设计逻辑仍停留在「表单填写」范式——用户需要手动选择日期、设置优先级、输入标签、填写描述。这套交互对熟悉工具的人尚可，但对新用户门槛较高，对碎片化场景（开会中、走路时、灵感一闪）尤其不友好。

### 1.2 核心痛点

| 编号 | 痛点 | 影响程度 |
|------|------|----------|
| P1 | 录入摩擦大：需要切换应用 → 新建任务 → 逐项填写字段 | 高 |
| P2 | 分类整理耗时：用户需要手动打标签、归类项目 | 中 |
| P3 | 任务堆积失控：没有智能推荐帮助用户决策「今天先做什么」 | 中 |
| P4 | 上下文丢失：任务描述简短，事后回看不知道当时在做什么 | 低 |

### 1.3 机会点

大语言模型可以理解用户的自然语言输入，自动提取任务的结构化信息（时间、优先级、标签等），将录入成本从「30 秒表单」压缩到「5 秒口语输入」。这是现有工具尚未很好解决的核心差异点。

## 2. 产品目标与成功指标

### 2.1 产品愿景

> 让每个人都能像和朋友说话一样管理自己的任务——零学习成本，AI 帮你搞定剩下的一切。

### 2.2 Phase 0（MVP）目标

- 核心目标：验证「自然语言录入」模式的用户体验可行性
- 关键结果 1：用户平均任务录入时间 ≤ 8 秒（vs 竞品 Todoist 平均 ~25 秒）
- 关键结果 2：AI 解析准确率（字段提取正确） ≥ 85%
- 关键结果 3：MVP 上线后 30 天内积累 100 名活跃用户

### 2.3 成功指标（KPI）

| 指标 | 定义 | 目标值（发布后 90 天） |
|------|------|----------------------|
| DAU | 每日活跃用户数 | ≥ 500 |
| 任务录入转化率 | 打开 App → 成功创建任务的比例 | ≥ 70% |
| AI 解析准确率 | 无需用户手动修正字段的比例 | ≥ 85% |
| D7 留存率 | 7 天后仍在使用的用户占比 | ≥ 40% |
| NPS | 净推荐值 | ≥ 30 |

## 3. 目标用户与用户画像

### 3.1 核心人群定位

- **主力用户**：25-40 岁，日常使用数字工具的知识工作者
- **特征**：信息密集型工作，任务碎片化，已有 Todo 工具使用习惯，对 AI 工具接受度高
- **平台偏好**：以 Web 端为主，习惯多设备切换

### 3.2 用户画像

#### Persona A：「高压产品经理」—— 小林，28 岁

- **职业**：互联网公司产品经理
- **典型场景**：全天开会，任务来自四面八方（Slack 消息、会议 Action Item、老板临时需求）
- **痛点**：
  - 开会中突然有新任务，没时间打开 App 认真填写
  - 任务太多，每天早上不知道先做哪个
  - 标签和分类总是混乱，找任务要翻很久
- **诉求**：
  - 随手一句话就能记下来，不用停下来整理
  - AI 帮我梳理今天应该优先搞什么
- **对 AI 的态度**：接受度高，已在日常使用 ChatGPT

#### Persona B：「多项目自由职业者」—— 阿南，34 岁

- **职业**：独立设计师，同时服务 3-5 个客户
- **典型场景**：每天在多个客户项目之间切换，任务来源分散（邮件、微信、自己的创意）
- **痛点**：
  - 客户需求通过微信发来，手动转移到 Todo 工具太麻烦
  - 截止日期管理混乱，容易漏掉小需求
  - 现有工具太「重」，每次记任务感觉在填表
- **诉求**：
  - 粘贴一段话，AI 帮我提取出任务和 deadline
  - 轻量、快速，不要有太多学习成本

#### Persona C：「习惯养成探索者」—— 晓雯，26 岁

- **职业**：刚入职的运营专员
- **典型场景**：工作和个人生活任务混在一起，想建立自己的任务管理系统
- **痛点**：
  - 试过 Todoist、滴答清单，但坚持不下去，操作太繁琐
  - 任务堆积后不知道如何消化，产生焦虑
  - 想要从任务管理工具中获得成就感
- **诉求**：
  - 低门槛，最好「打开就能用」
  - AI 帮我把模糊的目标拆解成具体步骤

## 4. 竞品分析

### 4.1 竞品对比矩阵

| 产品 | AI 能力 | 自然语言录入 | 价格 | 平台 | 核心定位 |
|------|---------|------------|------|------|---------|
| **Todoist** | 基础 NLP（日期解析） | 支持，仅解析时间 | 免费 / $4/月 | Web + 全平台 | 最成熟的个人任务管理 |
| **Things 3** | 无 | 不支持 | $49.99 买断 | macOS / iOS 专属 | 高颜值极简体验 |
| **Notion** | 基础 AI（需付费） | 不支持 | 免费 / $8/月 | Web + 全平台 | 万能文档 + 数据库 |
| **Reclaim.ai** | 强（日历智能调度） | 部分支持 | $8/月起 | Web | 时间块自动规划 |
| **Motion** | 强（任务自动排期） | 部分支持 | $19/月 | Web | AI 自动日历调度 |
| **TickTick（滴答）** | 中（日历视图 + 基础 AI）| 支持，仅时间 | 免费 / $28/年 | Web + 全平台 | 国内最成熟，功能全面 |
| **AI Todo（本品）** | 深度（全字段解析）| **核心能力，全字段** | TBD | Web（PWA） | 自然语言优先的轻量 Todo |

### 4.2 差异化机会

1. **录入体验差异**：现有工具的 NLP 仅解析日期/时间，本品解析全字段（标题、优先级、标签、描述、提醒）
2. **确认交互差异**：解析结果以「卡片预览」形式展示，用户一键确认或微调，透明可控
3. **轻量定位**：不做 Notion 式重量级工具，聚焦「快速记录 + 今日聚焦」，上手成本极低
4. **价格策略机会**：Reclaim/Motion 月费偏高，可用 Freemium 策略快速获取用户

### 5.2 F01 — 自然语言录入（P0，MVP 核心）

**功能描述**

用户在输入框中用自然语言描述任务，AI 自动解析并结构化，用户确认后创建任务。

**交互流程**

```
用户输入
  ↓
AI 实时解析（< 1.5s）
  ↓
解析结果预览卡片展示
  ↓
用户确认（一键）或手动微调
  ↓
任务创建成功
```

**解析字段**

| 字段 | 示例输入片段 | 解析结果 |
|------|------------|---------|
| 任务标题 | "和客户开会" | `和客户开会` |
| 截止日期 | "明天下午三点" | `2026-03-04 15:00` |
| 提醒时间 | "提前半小时提醒我" | `2026-03-04 14:30` |
| 优先级 | "优先级高" / "很紧急" | `P0` |
| 标签 | "关于项目 A" / "工作类" | `#项目A` |
| 描述补充 | "需要准备 PPT" | 追加到描述字段 |

**输入示例**

- `"明天下午三点提醒我和客户开会，优先级高，需要准备演示 PPT"`
- `"这周五之前要提交季度报告，很重要"`
- `"下午买牛奶，顺便取快递"`

**边界处理**

- 信息不足时（如只有 "去健身"），AI 创建基础任务，不强制要求补全字段
- 解析失败时，降级为手动填写表单，不阻断用户流程
- 支持粘贴一段话（如微信消息），AI 提取核心任务信息

---

### 5.3 F02 — 任务基础管理（P0）

**任务属性**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 任务名称，最大 200 字符 |
| 描述 | 富文本 | 否 | 详细说明，支持 Markdown |
| 截止日期 | 日期时间 | 否 | 精确到分钟 |
| 优先级 | 枚举 | 否 | P0（紧急）/ P1（高）/ P2（普通）/ P3（低） |
| 标签 | 多选文本 | 否 | 用户自定义标签 |
| 状态 | 枚举 | 是 | 待办 / 进行中 / 已完成 |

**视图**

| 视图名称 | 展示内容 | 优先级 |
|---------|---------|--------|
| 今日视图 | 今日截止 + 被标记为「今日聚焦」的任务 | P0 |
| 全部任务 | 所有任务，支持按状态/优先级/标签筛选 | P0 |
| 标签视图 | 按标签分组展示任务 | P1 |
| 已完成 | 完成任务的归档和回顾 | P1 |

**操作**

- 创建：通过自然语言输入框或手动表单
- 完成：点击复选框，支持快捷键 `Space`
- 编辑：点击任务进入详情页，逐字段编辑
- 删除：滑动删除或详情页删除，支持撤销（5 秒内）
- 拖拽排序：在列表中手动调整任务顺序

---

### 5.4 F03 — 账户与数据同步（P1）

- 注册/登录：邮箱 + 密码，支持 Google OAuth
- 跨设备同步：数据实时同步，延迟 < 3 秒
- 离线支持：PWA 模式，离线可查看任务，恢复网络后自动同步
- 数据导出：支持导出为 CSV / JSON 格式

---

### 5.5 F04 — 智能提醒（P1）

- 基于任务截止时间和用户设定的提醒时间发送通知
- 支持浏览器通知（Web Push）
- 提醒文案由 AI 生成，带有上下文（如"你今天还有 3 个任务未完成"）

---

### 5.6 F05 — AI 任务拆解（P2）

**功能描述**

用户输入一个模糊的大目标，AI 自动拆解为具体可执行的子任务列表。

**示例**

- 输入：`"准备季度述职报告"`
- 输出：
  - [ ] 收集本季度数据和 KPI 结果
  - [ ] 梳理主要成果和亮点
  - [ ] 整理遇到的问题和解决方案
  - [ ] 制作 PPT 框架
  - [ ] 完成 PPT 内容
  - [ ] 找同事 review

用户可以批量确认全部子任务，或手动增删调整。

---

### 5.7 F06 — 今日规划建议（P2）

**功能描述**

每天早晨，AI 根据用户所有任务的截止时间、优先级，推荐「今日应该完成的 3-5 个任务」，帮助用户聚焦。

**触发时机**

- 用户每日首次打开 App 时自动展示
- 可随时手动刷新建议

**建议逻辑（AI 提示词设计方向）**

- 优先推荐截止日期为今天/明天的任务
- 高优先级（P0/P1）优先展示
- 避免建议超过 5 个任务（避免用户焦虑）
- 附上简短理由（"这个明天截止，建议今天处理"）

---

### 5.8 F07 — 对话式任务查询（P2）

用户通过对话框查询任务信息，AI 理解意图后返回结果。

**示例查询**

- "本周还有哪些未完成的任务？"
- "帮我列出所有关于设计的任务"
- "我最近完成了什么？"

## 7. 非功能性需求

### 7.1 性能

| 指标 | 要求 |
|------|------|
| AI 解析响应时间 | P50 < 1.5s，P99 < 3s |
| 页面首次加载时间 | < 2s（FCP） |
| 任务列表渲染 | 支持 500+ 任务流畅滚动 |
| API 响应时间 | 普通接口 P50 < 200ms |

### 7.2 安全

- 所有数据传输使用 HTTPS/TLS 1.3
- 用户密码使用 bcrypt 加密存储
- AI 接口调用使用后端代理，不在前端暴露 API Key
- 定期安全审计，遵循 OWASP Top 10

### 7.3 可用性

- 服务可用性目标：99.5% uptime（月维度）
- 核心功能（任务 CRUD）需在 AI 服务不可用时降级可用

### 7.4 隐私

- 用户任务数据**不用于**模型训练或第三方分享
- 隐私政策页面清晰说明数据使用范围
- 支持用户申请删除所有数据（符合 GDPR/数据可携性要求）

### 7.5 可访问性

- 颜色对比度满足 WCAG AA 标准
- 核心操作支持纯键盘操作
- 支持屏幕阅读器（ARIA 标签）

## 8. 技术架构建议

### 8.1 技术栈推荐

| 层级 | 推荐技术 | 理由 |
|------|---------|------|
| 前端框架 | Next.js 14（App Router） | SSR/SSG 支持，Vercel 一键部署，生态成熟 |
| UI 组件库 | Tailwind CSS + shadcn/ui | 高定制性，开发效率高 |
| 后端框架 | Next.js API Routes | 全栈统一，减少维护成本 |
| AI 接入 | Anthropic Claude API（claude-sonnet-4-6） | 最强中英文理解能力，structured output 支持好 |
| 数据库 | PostgreSQL（via Supabase） | Supabase 提供开箱即用的 Auth + 实时同步 |
| 部署 | Vercel | 自动 CI/CD，全球 CDN，Next.js 原生支持 |
| 认证 | Supabase Auth / NextAuth.js | 支持 Email + Google OAuth，配置简单 |

### 8.2 AI 解析模块设计

**请求结构（伪代码）**

```
POST /api/ai/parse-task
Input: { raw_text: "明天下午三点提醒我和客户开会，优先级高" }

Output (Structured JSON):
{
  "title": "和客户开会",
  "due_date": "2026-03-04T15:00:00",
  "reminder_at": "2026-03-04T14:30:00",
  "priority": "P0",
  "tags": [],
  "description": "",
  "confidence": 0.95
}
```

**提示词设计原则**

- 使用 Claude 的 tool_use / structured output 确保 JSON 格式稳定
- System prompt 明确中文日期解析规则（"明天" 相对于当前时间计算）
- 加入 confidence 字段，低于阈值时提示用户手动确认

### 8.3 数据模型（核心表结构）

```sql
-- 用户表
users (id, email, created_at, ...)

-- 任务表
tasks (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  title TEXT NOT NULL,
  description TEXT,
  due_date TIMESTAMPTZ,
  reminder_at TIMESTAMPTZ,
  priority SMALLINT DEFAULT 2,  -- 0=P0, 1=P1, 2=P2, 3=P3
  status SMALLINT DEFAULT 0,    -- 0=待办, 1=进行中, 2=完成
  tags TEXT[],
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
)
```

## 9. MVP 范围与路线图

### 9.1 Phase 0 — MVP（目标：4-6 周）

**包含功能**
- ✅ 自然语言录入（AI 解析 + 预览确认）
- ✅ 任务 CRUD（创建、查看、完成、删除）
- ✅ 今日视图
- ✅ 全部任务视图（基础排序）
- ✅ 匿名使用（本地存储，不需要注册）

**不包含**
- ❌ 用户账户 & 云同步
- ❌ 标签系统
- ❌ 提醒通知
- ❌ AI 任务拆解

**验证目标**：100 个真实用户，收集录入体验和 AI 准确率数据

---

### 9.2 Phase 1（目标：MVP 后 4-6 周）

- ✅ 用户账户（邮箱 + Google OAuth）
- ✅ 跨设备数据同步
- ✅ 标签系统
- ✅ 优先级筛选
- ✅ 浏览器提醒通知
- ✅ PWA 支持

---

### 9.3 Phase 2（目标：Phase 1 后 6-8 周）

- ✅ AI 任务拆解
- ✅ 今日规划建议
- ✅ 对话式任务查询
- ✅ 数据导出（CSV / JSON）
- ✅ 已完成任务回顾视图

---

   │               │               │

## 10. 验收标准

### 10.1 F01 — 自然语言录入

| AC | 验收标准 |
|----|---------|
| AC-01-01 | 输入自然语言后，AI 解析结果在 1.5 秒内展示 |
| AC-01-02 | 解析卡片清晰展示：标题、截止时间、优先级、标签（如有） |
| AC-01-03 | 用户点击「确认创建」后，任务出现在任务列表中 |
| AC-01-04 | 用户可以点击任意解析字段进行修改后再确认 |
| AC-01-05 | 不包含时间的输入（如"去买菜"）可成功创建，仅有标题字段 |
| AC-01-06 | AI 服务超时或失败时，展示错误提示，并提供手动填写入口 |

### 10.2 F02 — 任务管理

| AC | 验收标准 |
|----|---------|
| AC-02-01 | 点击复选框后任务状态变为「已完成」，视觉上有划线效果 |
| AC-02-02 | 完成任务后 500ms 内从当前列表移出（或置灰，可配置） |
| AC-02-03 | 删除任务后出现 5 秒撤销提示，点击可恢复 |
| AC-02-04 | 今日视图只显示截止日期为今天的任务和手动标记为今日的任务 |
| AC-02-05 | 任务列表支持拖拽重新排序，松手后顺序持久化 |

### 10.3 F03 — 账户与同步（Phase 1）

| AC | 验收标准 |
|----|---------|
| AC-03-01 | Google OAuth 登录全程无需填写额外信息，小于 3 步完成 |
| AC-03-02 | 在设备 A 创建任务后，3 秒内在设备 B 可见 |
| AC-03-03 | 断网状态下可查看已缓存的任务列表 |
| AC-03-04 | 导出功能生成包含所有字段的 CSV 文件，可在 Excel 中正常打开 |
